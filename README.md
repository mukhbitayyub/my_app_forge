## 🛡️ Permissions Configuration

The application is structured to declare standard permissions under both Android and iOS configurations. By default, they are present but set up as standard templates:

### Android (`AndroidManifest.xml`)
All standard device permissions are grouped by category under `android/app/src/main/AndroidManifest.xml` and commented out. To enable a permission, uncomment its line:
```xml
<!-- CAMERA -->
<uses-permission android:name="android.permission.CAMERA"/>
```

### iOS (`Podfile` & `Info.plist`)
1. **Info.plist**: Under `ios/Runner/Info.plist`, each permission has a standard user description key:
   - `NSCameraUsageDescription`, `NSPhotoLibraryUsageDescription`, etc.
   - Update the description value string from the standard template `$(PRODUCT_NAME) requires access to your [X].` to match your application's actual context.
2. **Podfile**: Under `ios/Podfile`, we define the `GCC_PREPROCESSOR_DEFINITIONS` macro flags for the `permission_handler` pod. You can comment out or remove specific flags if your app does not require them to minimize App Store compliance checks.
